crosscloud.js (version !!VERSION!!)

This library makes it easy for browser apps to persist state, sharing with other instances of themselves and with compatible apps. The data lives on the user's pod, where it can be read and written by all applications (and other people) authorized by the user.

Some running examples:

hello A version of "Hello World" where you say hello to the whole world (or at least the other folks running this app, using a connected hub)
profile Simple user profile editor
contacts Simple contacts editor
ping Show how long it takes for data to be synchronized with the user's pod
drag Multi-user drag items around on the screen
login Shows current state of login/logout processing -- still in flux. Also shows app-selected skin.
autocomplete Shows completion of names from nearby in the users's social graph
personalsite Shows how one might do a "skin" for people (example depends on current test data)

Basic usage:

<script src="http://crosscloud.org/!!VERSION!!/crosscloud.js"></script>
<script src="https://code.jquery.com/jquery-2.1.1.min.js"></script>
<script>

var pod = crosscloud.connect();
pod.onLogin(function (userId) {

  pod.push({ ... some data item you want to add or update ...})
  // if it has no _id, then the item is created, and an _id will be added
  // if it has an _id, then the item will be updated

  var displayResults = function (results) {
     // 1. results will be a list of pages matching a query filter
     // 2. this function will be called every time the results change
  }

  pod.query()
     .filter({ someProperty: "its required value",
               anotherProperty: "its value",
               andAnother: { "$exists": true }  })
     .onAllResults(displayResults)
     .start();

});
</script>
